# Release Readiness Review

## 목적
- 결과물이 요구사항을 충족하는지 빠르게 판단한다.
- 배포 전 차단 이슈와 후속 개선 이슈를 분리한다.

## 입력
- 검수 대상 경로 또는 실행 방법
- 기대 동작(요구사항/완료 조건)
- 변경 범위(가능하면 커밋 또는 파일 목록)

## 절차
1. 요구사항을 검수 기준으로 변환한다.
2. 정상 경로를 먼저 검증한다.
3. 경계/오류 경로를 검증한다.
4. 실제 출력물(파일/로그/화면)을 근거로 이슈를 기록한다.
5. 배포 가능 여부를 `GO` 또는 `NO-GO`로 판정한다.

## 판정 규칙
- `NO-GO`: 데이터 손실, 핵심 기능 실패, 재현 가능한 크래시, 명백한 요구사항 불충족
- `GO with follow-ups`: 핵심 기능은 정상이나 개선 필요 항목 존재
- `GO`: 차단 이슈 없음

## 출력 형식
- `판정`: GO | GO with follow-ups | NO-GO
- `핵심 근거`: 1~3개
- `이슈 목록`: 심각도, 재현 절차, 기대 결과, 실제 결과, 영향 범위
- `후속 조치`: 담당 역할 기준 우선순위

## 작성 원칙
- 증거 없는 추정 표현을 피한다.
- 재현 불가 이슈는 별도로 분리한다.
- 모호한 표현 대신 관찰 가능한 사실을 적는다.
