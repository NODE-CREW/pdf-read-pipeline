# Improvement Roadmap

## 목적
- 개선 항목을 실행 가능한 단위로 쪼개고 순서를 정한다.
- 영향도 대비 비용이 좋은 항목부터 진행하게 만든다.

## 입력
- 현재 문제 목록(버그, 불편, 성능/품질 이슈)
- 제품 목표(예: 정확도, 속도, 안정성)
- 제약 조건(시간, 인력, 마감)

## 절차
1. 개선 항목을 수집하고 중복 항목을 통합한다.
2. 각 항목을 `Quick Win`, `Structural`, `Long-term`으로 분류한다.
3. 항목별 영향도와 구현 비용을 평가한다.
4. 우선순위를 정하고 2주/4주 실행 계획으로 배치한다.
5. 각 항목의 완료 조건(DoD)과 검증 방법을 명시한다.

## 우선순위 기준
- 사용자 영향도
- 실패/장애 리스크 감소 효과
- 구현 난이도와 선행 의존성
- 팀의 현재 처리 가능 용량

## 출력 형식
- `Top Priorities`: 바로 시작할 3개
- `2주 계획`: 작업, 기대 효과, 완료 기준
- `4주 계획`: 구조 개선 중심 작업
- `리스크`: 지연/품질 리스크와 완화책

## 작성 원칙
- 추상적 제안 대신 즉시 실행 가능한 작업 단위로 작성한다.
- 각 작업에 측정 가능한 성공 기준을 붙인다.
- 선행 작업이 필요한 항목은 순서를 명확히 고정한다.
